Gestion des Donnees Sensibles Filtrees

Valide Score: 97/100 Date: Decembre 2025

Resume

Implementation de la gestion des donnees sensibles filtrees cote serveur. L'application mobile affiche des placeholders appropries lorsque le serveur masque des informations selon le statut de l'utilisateur (non connecte, reservation non payee, etc.).

Comportement

3 Cas de Localisation

Cas Condition Affichage
1. Coordonnees exactes lat != null && longi != null Carte avec marker precis
2. Localisation approximative commune.nom != null Carte centree sur ville (sans marker)
3. Aucune localisation commune == null Placeholder cliquable

Placeholders par Contexte

Info Proprietaire .proprietaire() "Informations du proprietaire - Disponibles apres paiement"
Info Locataire .locataireAttente() "En attente de paiement" (vue proprio)

Utilisation

Afficher un placeholder de localisation

// Dans un widget
if (!address.hasExactLocation && !address.hasFallbackLocation) {
  return SensitiveDataPlaceholder.location(
    onTap: () => pushScreen(context, Reservation()),
  );
}

Verifier les infos proprietaire

final hasInfo = proprietaire?.hasSensitiveInfo ?? false;
if (!hasInfo) {
  return SensitiveDataPlaceholder.proprietaire(onTap: onMaskedTap);
}

Obtenir les coordonnees de fallback

// Retourne les coordonnees du centre de la commune/ville
final fallbackLocation = address.fallbackLocation;
final displayName = address.locationDisplayName;
