Amelioration Detail Appartement

Interface detail appartement et reservation optimisee pour la lisibilite

Description

Refonte complete de l'interface de detail des appartements et du processus de reservation. Cette feature ameliore la lisibilite en adoptant une palette monochrome (orange + gris), met en valeur le systeme de reductions pour sejours prolonges, et ajoute de nouvelles fonctionnalites pour les proprietaires (calendrier de blocage de dates).

Objectifs

  • Lisibilite amelioree - Reduction des couleurs excessives, palette coherente
  • Mise en valeur des reductions - Tableau des paliers visible, economie affichee en vert
  • Localisation approximative - Carte de zone pour les locataires avant paiement
  • Calendrier proprietaire - Blocage manuel de dates pour les proprietaires

Composants Crees

BaseLocationMap Nouveau
Widget generique pour afficher une carte de localisation. Gere 3 cas : localisation exacte (avec marker), localisation approximative (zone floue), et placeholder (aucune localisation). Utilise par ResidenceMapSection et AppartMapSection.
AppartMapSection Nouveau
Wrapper de BaseLocationMap specifique aux appartements. Extrait les coordonnees depuis la residence parente et gere l'affichage selon le role (locataire/proprietaire).
DetailSectionCard Nouveau
Card reutilisable pour encapsuler chaque section de la page detail. Fond surfaceColor (#2D2D2D), bordures arrondies 12px, titre optionnel avec icone.
AvailabilityCalendar Nouveau
Calendrier interactif pour les proprietaires permettant de visualiser et bloquer des dates. Legende couleurs : disponible (transparent), reserve (orange), bloque (gris).
AvailabilityBloc Nouveau
BLoC pour gerer l'etat des disponibilites. Events : LoadAvailability, BlockDates, UnblockDates, SelectDate, DeselectDate. States : AvailabilityLoaded, AvailabilityLoading, AvailabilityError.

Composants Modifies

RemiseInfo
Palette simplifiee : suppression des 4 couleurs (vert, teal, bleu, indigo) au profit d'une palette monochrome orange + gris. Nouveau parametre selectedDays pour surligner le palier actif avec bordure orange.
AppartBottom
Affichage enrichi du prix : prix original barre quand reduction applicable, badge pourcentage (-X%), ligne "Economisez X F" en vert (successColor #4CAF50). Utilise DiscountDetails.calculate().
AppartDetailContent
Reorganisation des sections : reductions juste apres le prix, ajout de la carte de localisation, suppression de la section "Info Proprietaire". Utilise DetailSectionCard pour le style.
ResidenceMapSection
Refactoring pour utiliser BaseLocationMap au lieu de dupliquer le code d'affichage de carte. Simple wrapper qui extrait les donnees de la residence.

Utilisation

Afficher la carte d'un appartement

AppartMapSection(
  appartement: appartement,
  isOwner: false,
  showExactLocation: false,  // true apres paiement
  onMaskedTap: () {
    // Action quand locataire tape sur zone masquee
  },
)

Afficher les reductions avec palier actif

RemiseInfo(
  remises: appartement.remises,
  prixBase: appartement.prix?.toDouble() ?? 0.0,
  selectedDays: nombreJours,  // Surligne le palier correspondant
)

Calendrier proprietaire

BlocProvider(
  create: (_) => AvailabilityBloc(),
  child: AvailabilityCalendar(
    appartementId: appartement.id!,
    isEditable: true,
  ),
)
