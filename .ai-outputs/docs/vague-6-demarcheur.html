🎯 Vue d'ensemble

La Vague 6 achève le rôle Démarcheur de l'app Asfar : 5 onglets fonctionnels couvrant le Dashboard, les Demandes, le tunnel « Nouvelle demande » en 3 étapes, le détail d'une référence avec timeline, le Wallet, et le Profile transverse partagé.

Particularité métier découverte au BA : un utilisateur peut cumuler plusieurs rôles (un démarcheur peut aussi être locataire ou propriétaire). Cette vague débloque le mécanisme de switch de rôle initié en Vague 5 mais resté inerte par dépendance.

« Un démarcheur peut avoir une interface de locataire, pareil pour un propriétaire » — décision BA actée 2026-05-09

  4. RoleHomeRouter.shellFor(user) retourne LocataireShell

  • Card propriétaire (réutilise HostCard Vague 5)

🎯 Décisions clés

1. Profile transverse

LocataireProfileScreen a été promu en ClientProfileScreen (déplacé dans shared/profile/) avec un helper ProfileDisplayInfo qui mappe user.type vers le subtitle, le badge et la liste de paramètres. Source : extras.jsx::Profile.profiles[role]. 

// Avant : 1 écran par rôle, hardcodé
// Après : 1 écran transverse, paramétré

ProfileDisplayInfo.forRole('demarcheur')
  → subtitle: 'Démarcheur · 27 clients'
  → badge: 'Top démarcheur'
  → settings: [Infos perso, Vérif, Méthode de retrait, Notifs, Préfs]

2. Switch de rôle muté en mémoire

Le tap dans le Role Switcher mute user.type directement puis appelle pushAndRemoveAll(RoleHomeRouter.shellFor(user)). Aucune persistance Hive en Vague 6 — c'est un cas connu, documenté dans la section TODO REBUILD de RECONSTRUCTION_UI_ASFAR.md pour la finition (event UserBloc::SetActiveRole).

3. Mocks (cohérence Vague 5)

Aucun BLoC réel branché en Vague 6. Les listes (référés, commissions, logements à pousser) viennent de SampleReferrals, SampleCommissions, SampleListingsToReferral. Pattern miroir de SampleListings (Vague 5). Les 22 BLoCs existants (DemarcheurBloc, PartenariatBloc, etc.) restent intacts pour branchement en vague de finition.

4. CTA « Retirer maintenant » → SnackBar stub

Le proto n'a pas défini la UX du retrait (bouton sans handler dans demarcheur.jsx:505). En Vague 6, le tap déclenche SnackBar('Retrait disponible prochainement'). La vraie UX sera spécifiée en F9 (Banque/Cartes/Compte) avec le contexte complet.

5. Tunnel NewReferralScreen single screen

Au lieu de 3 fichiers séparés (new_step1/2/3), un seul écran avec int _step. Pattern miroir de LocataireReserveScreen Vague 5 (293 lignes, 3 étapes). Avantages : header partagé, navigation forward/back triviale via setState, état du formulaire préservé sans param drilling. Écart documenté dans architecture.md § 3.2.

6. Empty states reportés

Le proto n'en a pas (toutes les listes mockées sont remplies). En Vague 6, les mocks sont toujours remplis donc cas non atteignable. Empty states à créer dans une vague de finition post-V9, brancher dans 6 zones (referrals vide, wallet historique, dashboard sections, locataire trips/favorites).
