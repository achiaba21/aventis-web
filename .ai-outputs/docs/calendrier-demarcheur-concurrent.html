📅 Calendrier Démarcheur — Plages Concurrentes

Gestion des plages EN_ATTENTE superposées avec visibilité concurrentielle des prix

Flutter BLoC Audit 94/100 v1.0 — 2026-03-09

🎯 Contexte & Objectif

Plusieurs démarcheurs peuvent soumettre des demandes de réservation (EN_ATTENTE) sur le même appartement pour les mêmes dates. La feature rend ces chevauchements visibles afin que chaque démarcheur puisse calibrer son offre de prix en connaissance de cause.

La règle métier clé : un démarcheur ne peut pas avoir deux demandes EN_ATTENTE pour le même jour sur le même appartement. S'il en a déjà une, le jour est verrouillé mais reste consultable.

🎨 Les 4 États d'un Jour

Cas A — Disponible

Aucune plage ou toutes DISPONIBLE.
Couleur : vert
Action : formulaire direct

Cas B — Occupé

Contient une plage OCCUPE.
Couleur : rouge
Action : non tappable

Cas C — Concurrence

EN_ATTENTE d'autres uniquement.
Couleur : orange
Action : bottom sheet + créer

Cas D — Ma demande

EN_ATTENTE dont une est la mienne.
Couleur : amber
Action : bottom sheet lecture seule

L'identification "ma plage" compare plage.demarcheurTelephone avec user.telephone après normalisation (suppression des espaces).

🔄 Flux Utilisateur

1
Le démarcheur ouvre le calendrier d'un appartement partenaire. Le mois courant est chargé via CalendarPlageBloc.
2
Chaque jour est coloré selon son cas (vert / orange / amber / rouge). Les jours avec EN_ATTENTE affichent des points sous le numéro (max 3 points, puis "N+").
3
Cas A : tap → formulaire de réservation pré-rempli avec la date.
4
Cas C : tap → bottom sheet avec la liste des concurrents (nom, téléphone, montant FCFA, durée). Bouton "Créer ma réservation" disponible.
5
Cas D : tap → même bottom sheet en lecture seule, sans bouton de création.
6
Après création d'une réservation, le calendrier est rechargé automatiquement.

⚙️ API — DayAnalysis

Helper pur (pas d'import Flutter/BLoC), instanciable à tout moment.

DayAnalysis({
  required List<CalendarPlage> plages,
  required String userTelephone,
})
PropriétéTypeDescription
casDayCasCas calculé : a / b / c / d
isTappableboolVrai si le tap doit être activé (A, C, D)
isBookableboolVrai si une nouvelle réservation est possible (A, C)
bgColorColorCouleur de fond de la cellule calendrier
headerColorColorCouleur du header dans le bottom sheet
badgeCountintNombre de plages EN_ATTENTE
hasBadgeboolVrai si badgeCount > 0
enAttentePlagesList<CalendarPlage>Plages EN_ATTENTE du jour

⚙️ API — CalendarPlagesLoaded.getPlagesForDay

List<CalendarPlage> getPlagesForDay(DateTime day)

Retourne toutes les plages couvrant le jour donné. Remplace les anciens getStatusForDay et getPlageForDay (supprimés).
