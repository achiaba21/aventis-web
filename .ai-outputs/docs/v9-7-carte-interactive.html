🗺️ Carte interactive — la première d'Asfar

Le proto V5-V8 ne reconstruisait qu'un MapTeaser figé (4 pins fake, image statique) sur le Home locataire. V9.7 livre la première vraie carte temps réel de l'app : le locataire peut désormais explorer visuellement les logements disponibles, taper un marker pour preview, voir ses propres positions GPS, et recharger la zone visible.

✅ Branché MapBloc ✅ Géoloc réelle ★ Tuiles OSM dark filtered /feature full pipeline

Vue d'ensemble

V9.7 introduit un nouvel écran LocataireMapScreen accessible via MapTeaser.onSeeMap du Home locataire (bouton « Voir carte »). L'écran consomme le MapBloc existant via LoadFilteredMapResidences(center, radiusKm) et orchestre :

  • Une carte flutter_map ^8.1.1 sur tuiles OSM filtrées en dark via ColorFiltered
  • Des markers prix compact (45k, 1.2M) accent or affichant les résidences disponibles
  • Un FAB « Ma position » qui demande la permission OS et recentre la carte sur le user (fallback Abidjan)
  • Un bouton « Rechercher dans cette zone » qui apparaît après pan/zoom et redispatche un load
  • Un BottomSheet au tap marker affichant un preview + bouton « Voir détails » → push LocataireDetailScreen
  • Un bouton « Filtrer » top-right qui ouvre LocataireSearchScreen (filtres existants V5)
  • 3 overlays partagés (Loading / Empty / Error) via MapOverlayCard

Parcours utilisateur

Flow principal — exploration carte

  1. Locataire tape « Voir carte » dans le MapTeaser du Home (section « Près de vous »)
  2. L'écran LocataireMapScreen push, dispatche LoadFilteredMapResidences(center=Abidjan, radius=10km) en initState
  3. Tuiles OSM filtrées dark s'affichent, état MapLoading overlay petite card centrée
  4. État MapResidencesLoaded → markers prix accent or apparaissent sur les positions displayLat/displayLongi
  5. Utilisateur tape un marker → showModalBottomSheet(MapMarkerBottomSheet) : image placeholder 16:9, titre, prix, communeName, bouton « Voir détails »
  6. Tap « Voir détails » → mapping MapResidence → ListingPreview → pushScreen(LocataireDetailScreen)

Flow géolocalisation

  1. Tap FAB « Ma position » → spinner intégré dans le FAB
  2. LocationUtil.getCurrentLatLng() demande la permission OS au premier tap
  3. Si accordée → mapCtrl.move(userPosition, zoom=14) + reload zone
  4. Si refusée → SnackBar « Activez la géolocalisation dans les paramètres pour utiliser cette fonction »

Flow recherche zone visible

  1. User pan/zoom la carte → MapEventMoveEnd émis
  2. _currentCenter mis à jour depuis mapCtrl.camera.center
  3. Bouton « Rechercher dans cette zone » apparaît via AnimatedOpacity 200ms
  4. Tap → LoadFilteredMapResidences(newCenter, radius=10km) + bouton disparaît

Flow filtres

  1. Tap icône « tune » top-right de l'AppBar
  2. pushScreen(LocataireSearchScreen) — l'écran de filtres existant V5
  3. Au retour, le user peut taper « Rechercher dans cette zone » pour appliquer les nouveaux filtres

Décisions techniques actées

1. Tuiles OSM dark via tileBuilder

Le défi : les tuiles OSM par défaut sont en thème light, contraste fort avec la palette Asfar dark. La solution retenue applique un ColorFiltered avec matrice invert+désaturation uniquement sur les tuiles (pas sur les markers) via le tileBuilder de TileLayer :

TileLayer(
  urlTemplate: 'https://tile.openstreetmap.org/{z}/{x}/{y}.png',
  tileBuilder: (context, tileWidget, tile) {
    return ColorFiltered(
      colorFilter: const ColorFilter.matrix(_darkenMatrix),
      child: tileWidget,
    );
  },
)

Conséquence : zéro dépendance externe, zéro clé API, markers accent or préservés en couleur native. Si les utilisateurs préfèrent un thème light plus tard, c'est trivialement désactivable.

2. Pas de clustering en V9.7

Markers individuels uniquement. L'event LoadClusteredMapResidences du MapBloc existe → activable en V10 sans refactor majeur si le besoin émerge (par exemple zones très denses).

3. Géoloc via LocationUtil existant

Découverte d'audit : LocationUtil existait déjà dans lib/util/ avec exactement les helpers nécessaires (getCurrentLatLng, checkAndRequestPermissions). Aucun LocationService dupliqué n'a été créé — principe DRY appliqué.

4. Pas de nouvel event MapBloc

L'écran consomme uniquement LoadFilteredMapResidences (center, radiusKm, filter). Les états MapLoading/ResidencesLoaded/Error/Empty/Selected existent déjà. Zéro modification du BLoC = zéro régression possible sur l'infra.

5. Pas d'onglet BottomNav « Map »

Le LocataireShell a 5 onglets (Explorer/Voyages/Favoris/Messages/Profil). Ajouter un 6ème surchargerait la barre. L'écran est pushé depuis MapTeaser.onSeeMap du Home — point d'entrée unique cohérent avec le pattern proto.

6. Refactor DRY pendant l'audit

L'auditeur a détecté un pattern dupliqué (Container bgElev1.95 + line + radius lg + shadow blur 12) répété 3 fois dans les overlays Loading/Empty/Error. Extraction immédiate dans MapOverlayCard :

  • Avant : 47+44+45 = 136 lignes pour les 3 overlays
  • Après : 33+24+26+45 = 128 lignes (-8 lignes nettes + 1 widget partagé)
  • Score DRY : 90 → 100 (+10)

Limites V9.7 actées

  • Pas de clustering markers — l'event LoadClusteredMapResidences est disponible mais non câblé. Activation V10 si besoin.
  • Pas de search-as-you-type — l'utilisateur doit explicitement taper « Rechercher dans cette zone » après pan/zoom.
  • Pas de retour automatique des filtres LocataireSearchScreen — au retour, le user doit redéclencher un load manuellement.
  • iOS Simulator sans GPS → fallback Abidjan systématique. Tester sur device réel.
  • Tile loading 2-3s sur 3G/4G — non-bloquant (markers attendent les tuiles).
  • Résidences sans coordonnées (displayLat/Longi null) — exclues silencieusement.
